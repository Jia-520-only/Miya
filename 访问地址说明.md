
## 🎯 重要提示

**前端端口是 3000，不是 5173！**

这是因为在 `vite.config.ts` 中配置的端口是 3000。

## ⚠️ 注意事项

1. **首次访问可能需要等待 3-5 秒**让前端完全加载
2. **端口 3000 已被占用**会显示连接错误，请关闭其他使用该端口的程序
3. **数据库连接失败**是正常的（Milvus、Neo4j、Redis 未运行），会使用模拟模式

## 🔧 如果仍然无法访问

### 方案 1: 直接访问 API
```
http://localhost:8000/health
```
如果这个能访问，说明后端正常，问题在前端。

### 方案 2: 手动启动前端
```batch
cd miya-pc-ui
npm run dev:web
```

### 方案 3: 检查防火墙
确保 Windows 防火墙允许端口 3000：

```powershell
New-NetFirewallRule -DisplayName "Miya Web UI" -Direction Inbound -Protocol TCP -LocalPort 3000 -Action Allow
```
