# 记忆检索功能使用指南

## 功能概述

弥娅现在可以自主从普通记忆和游戏记忆中检索信息并汇报给用户。

## 可用工具

### 1. `search_normal_memory` - 检索普通记忆

**功能**: 检索普通模式下的对话历史记忆

**参数**:
- `limit` (可选): 返回数量限制,默认10条
- `days_back` (可选): 检索最近几天的记忆,默认7天
- `keywords` (可选): 关键词列表,用于筛选

**使用场景**:
- 用户询问"我们之前聊过什么?"
- 用户询问"我上周说了什么?"
- 用户询问"关于XX话题我们聊过什么?"

**示例**:
```
用户: 弥娅,我们最近聊过什么?
弥娅: 让我检索一下记忆...

## 普通记忆检索报告

**摘要**: 找到5条最近对话记忆，涉及3位用户。

**详细内容**:

1. 你昨天说今天要考试 (时间: 2026-02-28T10:30:00)
2. 你问过我关于Python的学习建议 (时间: 2026-02-27T15:20:00)
3. 你分享了一首好听的歌曲 (时间: 2026-02-26T20:15:00)
4. 你说周末想去爬山 (时间: 2026-02-25T18:45:00)
5. 你问了我喜欢吃什么 (时间: 2026-02-24T12:00:00)

嗯,从记忆来看,你最近在准备考试,还想周末去爬山呢!
```

### 2. `search_game_memory` - 检索游戏记忆

**功能**: 检索游戏模式下的记忆,包括角色卡、故事进度、存档信息等

**参数**:
- `limit` (可选): 返回数量限制,默认10条
- `keywords` (可选): 关键词列表,用于筛选

**使用场景**:
- 用户询问"我的角色卡是什么?"
- 用户询问"故事进展到哪了?"
- 用户询问"游戏中发生了什么?"

**示例**:
```
用户: KP,我的角色卡是什么?
弥娅: 让我检索一下游戏记忆...

## 游戏记忆检索报告

**摘要**: 找到3条游戏记忆，1个角色，故事进度。

**详细内容**:

1. 角色: 黄泉 (玩家: 黄泉)
   属性: {'力量': 60, '敏捷': 70, '体质': 50, '智力': 80, '教育': 75, '外貌': 90}
   技能: {'侦查': 70, '聆听': 65, '图书馆': 75, '神秘学': 60}
   时间: 2026-02-28T15:00:00

2. 故事进度: {'location': '废弃精神病院', 'current_scene': '大厅探索', 'characters_met': ['守门人']}
   时间: save_001

3. 存档: 第一章 (ID: save_001)
   时间: 2026-02-28T16:00:00

你的角色叫黄泉,力量60,敏捷70...目前你们正在废弃精神病院的大厅里探索呢!
```

## 工具调用机制

弥娅会根据用户的自然语言自动判断是否需要检索记忆:

### 触发条件
1. **时间相关**: "最近"、"上周"、"前几天"等时间词
2. **记忆相关**: "聊过"、"说过"、"记得"、"忘记"等记忆词
3. **游戏相关**: "角色卡"、"故事"、"进度"、"属性"等游戏词
4. **直接询问**: "我们聊过什么?"、"我的角色是什么?"等直接问题

### 自动判断逻辑
- 如果提到时间词 → 调用 `search_normal_memory`
- 如果提到游戏相关词且在游戏模式 → 调用 `search_game_memory`
- 如果直接询问历史 → 调用 `search_normal_memory`

## 记忆存储位置

### 普通记忆
- **存储位置**: `data/conversation_history/`
- **管理器**: `MemoryNet`
- **内容**: 普通对话历史

### 游戏记忆
- **存储位置**: `data/game_memory/groups/{group_id}/games/{game_id}/`
- **管理器**: `GameMemoryManager`
- **内容**: 角色卡、故事进度、存档数据

## 注意事项

1. **权限控制**:
   - 普通记忆检索: 所有用户都可以检索自己的记忆
   - 游戏记忆检索: 可以看到公开的角色卡和自己的角色卡

2. **时间限制**:
   - 普通记忆检索默认只检索最近7天
   - 可以通过 `days_back` 参数调整

3. **数量限制**:
   - 默认返回10条记忆
   - 可以通过 `limit` 参数调整

4. **关键词筛选**:
   - 可以提供多个关键词
   - 只要内容包含任一关键词就会被检索到

## 示例对话

### 场景1: 普通记忆检索

```
用户: 弥娅,我上周跟你说什么了?
弥娅: 让我检索一下记忆...

[检索并汇报记忆]

嗯,你上周主要跟我聊了考试准备的事情呢!
```

### 场景2: 游戏记忆检索

```
用户: KP,我想知道我角色的属性
弥娅: 让我查看一下角色卡...

[检索并汇报角色卡]

你的力量是60,敏捷是70...还算不错的配置呢!
```

### 场景3: 混合查询

```
用户: 弥娅,我们关于考试聊过什么?
弥娅: (调用 search_normal_memory, keywords=["考试"])

[检索并汇报相关记忆]

关于考试,我们聊过...
```

## 技术细节

### 文件结构
```
webnet/EntertainmentNet/game_mode/
├── memory_retriever.py          # 记忆检索核心类
├── memory_search_tools.py       # 记忆检索工具函数
└── tools/
    ├── search_normal_memory.py  # 检索普通记忆工具
    └── search_game_memory.py    # 检索游戏记忆工具
```

### 依赖关系
- `MemoryRetriever`: 核心检索类
- `MemoryNet`: 普通记忆管理器
- `GameModeManager`: 游戏记忆管理器

### 异步处理
- 所有检索操作都是异步的
- 使用 `asyncio` 确保不阻塞主流程
