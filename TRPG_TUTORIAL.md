## TRPG 是什么

TRPG (Tabletop Role-Playing Game) 是桌面角色扮演游戏，玩家通过角色扮演，在虚构的世界中进行冒险。

与传统游戏不同，TRPG 没有固定的结局和画面，一切都在玩家的想象和规则框架中展开。

### 核心要素
- **KP (Keeper / DM)**: 游戏主持人，负责描述场景、扮演 NPC、裁定规则
- **PC (Player Character)**: 玩家角色，由玩家操作的角色
- **NPC (Non-Player Character)**: 非玩家角色，由 KP 扮演
- **骰子**: 随机性工具，决定行动成功与否
- **规则书**: 游戏的规则框架

## 基础概念

### 骰子表示法

| 表示法 | 含义 | 举例 |
|--------|------|------|
| `d6` | 1 个六面骰 | 1d6 = 1-6 |
| `2d6` | 2 个六面骰，求和 | 2d6 = 2-12 |
| `d100` | 百分骰 | 1d100 = 1-100 |
| `3d6+2` | 3 个六面骰加 2 | 3d6+2 = 5-20 |

### 难度等级

**COC7 难度**
- **常规难度**: 技能值 ≤ 结果 → 失败
- **困难难度**: 技能值/2 ≤ 结果 → 失败
- **极难难度**: 技能值/5 ≤ 结果 → 失败

**D&D5E 检定**
- 轻松任务: DC 5
- 简单任务: DC 10
- 普通任务: DC 15
- 困难任务: DC 20
- 极难任务: DC 25+

## COC7 规则简介

### 核心机制

COC7 (Call of Cthulhu 7th Edition) 是克苏鲁的呼唤第七版规则，以恐怖和调查为核心。

#### 属性 (STR, DEX, POW, CON, APP, EDU, SIZ, INT)

每个角色有 8 个基本属性，范围 0-100。

#### 技能检定

```
/skill_check <技能名> [难度] [修正值]
```

示例：
```
/skill_check 观察 常规
/skill_check 潜行 困难 +10
```

#### 大成功/大失败

- **大成功**: 骰出 1，获得额外成功
- **大失败**: 骰出 100，严重失败
- **极难成功**: 低于技能值/5
- **困难成功**: 低于技能值/2
- **常规成功**: 低于技能值

#### 理智 (SAN) 检定

角色遇到恐怖事物需要进行理智检定：
```
/skill_check 理智 [难度]
```

理智损失通常在 `1/1d6` 到 `1d10/1d100` 之间。

## D&D5E 规则简介

### 核心机制

D&D5E (Dungeons & Dragons 5th Edition) 是龙与地下城第五版规则，以战斗和冒险为核心。

#### 属性 (力量、敏捷、体质、智力、感知、魅力)

每个属性有 8-20 的数值，对应修正值 `-1` 到 `+5`。

#### 技能检定

```
/skill_check <技能名> [修正值]
```

示例：
```
/skill_check 说服 +3
/skill_count 奥秘 +5
```

#### 优势/劣势

- **优势**: 骰 2 次，取高值
- **劣势**: 骰 2 次，取低值

#### 战斗

```
/attack <武器名> [修正值]
/combat_log
```

## 如何开始游戏

### 第一步：创建 TRPG 会话

在群聊中输入：
```
/start_trpg [规则系统]
```

示例：
```
/start_trpg coc7
/start_trpg dnd5e
```

### 第二步：设置 KP (游戏主持人)

```
/kp set_kp @KP用户
```

### 第三步：玩家创建角色卡

```
/create_pc <角色名> <属性...>
```

示例：
```
/create_pc 爱丽丝 60 50 60 50 70 80 50 60
```

查看角色：
```
/show_pc <角色名>
```

## 游戏流程

### 典型跑团流程

```
1. KP 描述场景
   ↓
2. PC 提出行动
   ↓
3. KP 判定是否需要检定
   ↓
4. PC 进行检定
   ↓
5. KP 描述结果
   ↓
6. 重复 2-5
```

### 示例：调查废弃公寓

**KP**: "你们来到了一间废弃的公寓，门虚掩着，里面一片漆黑。"

**PC 玩家**: "我想推开门，用技能观察里面有什么。"

**KP**: "需要做一个观察检定。"

**PC**: `/skill_check 观察`

**系统**: "观察检定: 骰出 45 (技能值 60) → 成功"

**KP**: "你的观察力很强，注意到角落里有一个人影，似乎在盯着你们。"

## 角色创建

### COC7 角色卡

```
/create_pc 爱丽丝 STR DEX POW CON APP EDU SIZ INT
```

属性范围：
- STR (力量): 体力相关
- DEX (敏捷): 灵巧度
- POW (意志): 精神力
- CON (体质): 耐久力
- APP (外貌): 魅力
- EDU (教育): 知识水平
- SIZ (体型): 身材大小
- INT (智力): 智商

### 更新角色卡

```
/update_pc 爱丽丝 观察 80
/update_pc 爱丽丝 理智 70
```

### 删除角色卡

```
/update_pc 爱丽丝 删除
```

## KP 和 PC 的职责

### KP 的职责

1. **场景描述**: 用生动的语言描述环境
2. **规则裁定**: 根据规则书判定行动结果
3. **NPC 扮演**: 扮演所有非玩家角色
4. **故事推进**: 推动剧情发展，保持节奏
5. **公平公正**: 保持游戏的公平和趣味性

### PC 的职责

1. **角色扮演**: 扮演好自己的角色
2. **团队协作**: 与其他玩家合作
3. **主动探索**: 积极提出行动方案
4. **遵守规则**: 尊重规则和 KP 的裁定
5. **享受游戏**: 保持轻松愉快的心态

## 实战技巧

### 给 KP 的建议

#### 1. 场景描述要生动
```
❌ 不好: "你们看到一间房间。"
✅ 好: "推开沉重的橡木门，霉味扑面而来。昏暗的灯光下，你看到房间中央有一张破旧的桌子，桌上散落着发黄的文件。墙上挂着一张褪色的油画，画中人的眼睛似乎在盯着你。"
```

#### 2. 不要直接说"失败"
```
❌ 不好: "你失败了，什么也没发现。"
✅ 好: "你仔细搜查了整个房间，但一无所获。不过，你注意到地毯下似乎有什么东西。"
```

#### 3. 给玩家选择
```
✅ 好: "你可以选择: 1) 强行破门 2) 潜行进去 3) 先在周围调查"
```

#### 4. 合理使用暗骰
对于需要保密的信息，使用 `/secret` 暗骰：
```
/secret 1d100
```
结果会私聊发送给 KP。

### 给 PC 的建议

#### 1. 明确描述你的行动
```
❌ 不好: "我检定一下。"
✅ 好: "我想用观察技能看看桌上有什么。"
```

#### 2. 善用技能
不要只用战斗技能，合理使用观察、说服、调查等技能。

#### 3. 团队合作
不同角色有不同优势，分工合作解决问题。

#### 4. 接受失败
TRPG 的魅力在于不确定性，失败也是故事的一部分。

## 进阶主题

### KP 模式

系统支持三种 KP 模式：

1. **independent (独立)**: 每个 KP 只能管理本团的会话
2. **cross_group (跨群)**: 一个 KP 可以管理多个群
3. **global (全局)**: 只有一个全局 KP

设置模式：
```
/kp set_mode cross_group
```

### 暗骰功能

对于需要保密的信息，KP 可以使用暗骰：
```
/secret 1d100
```

只有 KP 会收到结果，适合用于：
- 隐藏信息
- 惊吓效果
- 剧情伏笔

### 线索系统

KP 可以添加线索供玩家查看：
```
/kp add_clue 发现一张旧照片
/kp list_clue
```

### NPC 管理

添加和管理 NPC：
```
/kp add_npc 守卫队长
/kp list_npc
```

## 常见问题

### Q: 骰点结果不好怎么办？
A: 接受结果，想办法解决。失败的行动往往是故事的转折点。

### Q: KP 的裁决不公怎么办？
A: 礼貌地提出疑问，参考规则书。最终尊重 KP 的决定。

### Q: 如何成为好的 KP？
A: 多准备、多练习、保持公平、让每个玩家都有参与感。

### Q: 如何创造有趣的故事？
A: 从简单开始，根据玩家的选择推进，保持灵活性和创造性。

### Q: 可以中途更换规则吗？
A: 不建议，规则系统会影响游戏平衡。如有需要，最好在新团使用。
