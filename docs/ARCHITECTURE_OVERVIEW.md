## 架构设计理念

### 1. 蛛网式分布式架构

弥娅采用蛛网式（Web-like）分布式架构，模仿自然界蛛网的结构特点：

```
        核心层 (Center)
           │
    ┌──────┼──────┐
    │      │      │
  中枢层  传输层  感知层
    │      │      │
    └──────┼──────┘
           │
        子网层
```

**设计优势：**
- 🕸️ **去中心化**：各模块相对独立，可独立演化
- 🔗 **强韧性**：部分节点故障不影响整体
- 🚀 **高扩展**：新增子网/模块无需重构核心
- 📡 **多路径**：数据可通过多条路径传输

### 2. 五流传输机制 (M-Link)

M-Link定义了五种数据流，确保系统各部分有序通信：

| 流类型 | 代码方向 | 作用 | 优先级 |
|-------|---------|------|-------|
| 指令流 | 下行 | 内核/中枢 → 执行节点 | 最高 |
| 感知流 | 上行 | 感知层 → 中枢/子网 | 高 |
| 同步流 | 横向 | 子网 ↔ 子网 | 中 |
| 信任流 | 双向 | 信任评分与传播 | 低 |
| 记忆流 | 双向 | 记忆读写请求 | 中 |

### 3. 动态人格演化

人格不是静态配置，而是可演化的系统：
- 对话会微调人格数值
- 记忆会强化特定特质
- 情绪会影响人格表现
- 约束保证人格不偏离边界

### 4. GRAG记忆架构

采用 **G**raph-**R**etrieval-**A**ugmented-**G**eneration：
- **五元组**：主体-动作-对象-上下文-时间
- **知识图谱**：Neo4j存储关系网络
- **向量检索**：Milvus提供语义搜索
- **动态重组**：语义引擎持续重组记忆

## 分层详解

### 1. 用户界面层 (UI Layer)

**职责：** 用户交互入口，将用户输入转换为系统可理解的消息

**模块：**

#### 命令行 (CLI)
- 文件：`run/main.py`
- 功能：基础对话交互
- 特点：简单、直接

#### PC 端 (PC UI)
- 文件：`pc_ui/`
- 技术：Electron + WebUI
- 功能：
  - 对话系统
  - 群聊系统
  - 笔记管理
  - 媒体控制
  - 插件管理
  - 系统监控

#### QQ 机器人 (QQ Bot)
- 文件：`run/qq_main.py`, `webnet/qq.py`
- 协议：OneBot
- 功能：QQ群聊/私聊接入

**职责：** 提供RESTful API和运行时管理能力

## 核心模块

### 人格系统 (Personality)

**五维人格向量：**

```python
{
    'warmth': 0.8,      # 温暖度
    'logic': 0.7,       # 逻辑性
    'creativity': 0.6,  # 创造力
    'empathy': 0.75,    # 同理心
    'resilience': 0.7   # 韧性
}
```

**边界约束：**
```python
{
    'min_warmth': 0.3,
    'max_warmth': 1.0,
    'min_logic': 0.4,
    'max_logic': 0.9
    # ...其他维度
}
```

**稳定性计算：**
```python
stability = 1.0 - variance(vectors)
```

### 情绪系统 (Emotion)

**情绪类型：**
- 积极情绪：joy, excitement, satisfaction, relief, gratitude
- 消极情绪：sadness, anxiety, anger, loneliness, confusion
- 中性情绪：calm, curiosity, focus, thinking, patience

**情绪染色：**
```python
def influence_response(response):
    # 根据情绪调整回复
    if emotion == 'joy':
        return "😊 " + response
    elif emotion == 'sadness':
        return "😔 " + response
    # ...
```

### 记忆系统 (Memory)

**GRAG架构：**
```
(主体) -[:PERFORMED]-> (动作) -[:ON]-> (对象)
   │                             │
   └──────[:IN_CONTEXT]──────────┘
              ↓
           (上下文)
              │
              [:AT_TIME]
              ↓
           (时间)
```

**检索策略：**
1. 语义向量搜索
2. 关键词匹配
3. 时间窗口过滤
4. 上下文相关性

## 数据流

### 对话流程

```
用户输入
   ↓
[UI层] 捕获输入
   ↓
REST API / WebSocket
   ↓
[应用层] RuntimeAPI Server
   ↓
[M-Link] 发送感知流
   ↓
[感知层] PerceptualRing → AttentionGate
   ↓
[中枢层] MemoryEngine (检索记忆)
   ↓
[中枢层] Emotion (情绪染色)
   ↓
[中枢层] Decision (决策)
   ↓
[核心层] PromptManager (构建提示词)
   ↓
[LLM] 生成回复
   ↓
[中枢层] Emotion (情绪更新)
   ↓
[记忆层] 存储对话
   ↓
[M-Link] 返回响应流
   ↓
[UI层] 显示消息
```

### 记忆流程

```
对话发生
   ↓
[记忆层] QuintupleExtractor (提取五元组)
   ↓
   ├─→ [存储] Redis (潮汐记忆, TTL=1h)
   ├─→ [存储] Milvus (向量记忆)
   └─→ [存储] Neo4j (知识图谱)
   ↓
[语义层] SemanticDynamicsEngine (重组语义)
   ↓
[记忆层] VectorCache (缓存)
```

### 情绪流程

```
对话触发
   ↓
[情绪层] Emotion.influence_coloring()
   ↓
   ├─→ 影响回复语气
   ├─→ 影响决策权重
   └─→ 影响记忆权重
   ↓
[情绪层] Emotion.decay_coloring()
   ↓
(情绪自然衰减)
```
