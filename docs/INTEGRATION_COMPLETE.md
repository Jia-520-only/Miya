## 🔧 配置说明

### 高级能力配置文件

位置: `config/advanced_config.json`

```json
{
  "advanced_capabilities": {
    "enabled": true,
    "enable_task_planning": true,
    "enable_autonomous_exploration": true,
    "enable_intelligent_execution": true,
    "enable_chain_of_thought": true,
    ...
  }
}
```

### 关键配置项

- `enabled`: 是否启用高级能力（默认: true）
- `max_steps`: 最大探索步数（默认: 50）
- `max_concurrent_tasks`: 最大并发任务数（默认: 3）
- `enable_rollback`: 是否启用回滚（默认: true）
- `enable_result_validation`: 是否启用结果验证（默认: true）

## 📝 工作流程

```
用户输入复杂任务
    ↓
DecisionHub 接收输入
    ↓
检测是否为复杂任务 (_should_use_advanced_orchestration)
    ↓
是 → 启动高级编排器
    ↓
1. 思维链分析（理解目标）
    ↓
2. 任务规划（分解为子任务）
    ↓
3. 主动探索（如果需要）
    ↓
4. 执行任务（智能执行器）
    ↓
5. 反思和总结（改进）
    ↓
返回格式化的结果
    ↓
情绪染色
    ↓
返回给用户
```

## 🎯 核心突破

### 1. 自主性
- **之前**: 被动响应，只能执行用户明确指定的命令
- **现在**: 主动探索环境，自主收集信息

### 2. 规划能力
- **之前**: 线性执行工具调用，最多20次迭代
- **现在**: 智能规划任务分解，支持依赖关系

### 3. 可靠性
- **之前**: 单个错误就会导致整个任务失败
- **现在**: 支持重试、回滚、事务管理

### 4. 可解释性
- **之前**: 黑盒执行，用户不知道AI在做什么
- **现在**: 结构化思维链，每一步都透明可追溯

## ⚠️ 注意事项

### 性能考虑
- 高级编排会增加响应时间（5-20秒）
- 对于简单任务不会启动高级编排
- 建议根据实际情况调整配置参数

### 成本考虑
- 频繁调用AI会增加API成本
- 已启用缓存机制减少重复计算
- 可调整探索步数控制成本

### 错误处理
- 高级编排失败会自动降级到普通处理
- 详细日志记录便于调试
- 不会影响弥娅的正常功能
