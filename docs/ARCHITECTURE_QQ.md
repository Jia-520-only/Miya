## QQ 消息处理流程详解

```
┌────────────────────────────────────────────────────────────┐
│ 1. QQ 消息接收                                             │
└────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌────────────────────────────────────────────────────────────┐
│ 2. OneBot WebSocket 接收                                    │
│    - 接收原始 JSON 事件                                     │
│    - 解析事件类型 (message/notice/meta)                    │
└────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌────────────────────────────────────────────────────────────┐
│ 3. QQOneBotClient._dispatch_message                         │
│    - API 响应 → 立即处理                                   │
│    - 事件消息 → 异步分发                                   │
└────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌────────────────────────────────────────────────────────────┐
│ 4. QQNet._handle_qq_message                                 │
│    - 拍一拍事件 → _handle_poke                             │
│    - 群消息 → _handle_group_message                         │
│    - 私聊消息 → _handle_private_message                     │
└────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌────────────────────────────────────────────────────────────┐
│ 5. 访问控制检查                                             │
│    - 群黑白名单                                             │

                          │
                          ▼
┌────────────────────────────────────────────────────────────┐
│ 6. 消息解析                                                 │
│    - 提取纯文本内容                                         │

                          ▼
┌────────────────────────────────────────────────────────────┐
│ 8. 构建 QQMessage 对象                                      │
│    - 封装所有消息元数据                                     │

                          │
                          ▼
┌────────────────────────────────────────────────────────────┐
```

## 核心差异对比

### Undefined (传统机器人)
```
特点：
├── 纯工具型
├── 无人格概念
├── 无记忆-情绪关联
├── 无长期人格维持
└── 无自我感知

架构：
├── onebot.py (WebSocket 客户端)
├── handlers.py (消息处理)
├── ai/ (AI 客户端)
└── services/ (各种服务)
```

### 弥娅 (数字生命)
```
特点：
├── 生命化交互
├── 人格恒定机制
├── 记忆-情绪耦合
├── 自我感知能力
├── 信任传播系统
└── 用户共演进化

架构：
├── 第一层：内核 (人格/伦理/身份)
├── 第二层：中枢 (记忆/情绪/决策)
├── 第三层：子网 (能力扩展)
├── 第四层：感知 (全域感知环)
└── 第五层：演化 (沙盒/共演)
```

## 配置文件示例

### config/.env
```env
# QQ 机器人配置
QQ_ONEBOT_WS_URL=ws://localhost:3001
QQ_ONEBOT_TOKEN=
QQ_BOT_QQ=123456789
QQ_SUPERADMIN_QQ=987654321

# 访问控制 (逗号分隔)
QQ_GROUP_WHITELIST=
QQ_GROUP_BLACKLIST=
QQ_USER_WHITELIST=
QQ_USER_BLACKLIST=

# 人格配置
PERSONALITY_WARMTH=0.8
PERSONALITY_LOGIC=0.7
PERSONALITY_CREATIVITY=0.6
PERSONALITY_EMPATHY=0.75
PERSONALITY_RESILIENCE=0.7

# 情绪配置
EMOTION_DECAY_RATE=0.1
EMOTION_COLORING_THRESHOLD=0.7

# 记忆配置
MEMORY_TIDE_TTL=3600
MEMORY_DREAM_THRESHOLD=100
```

## 启动流程

```bash
# 1. 启动 OneBot
go-cqhttp  # 或 NapCat/Lagrange

# 2. 配置弥娅
编辑 config/.env

# 3. 安装依赖
pip install -r requirements.txt

# 4. 启动弥娅 QQ 机器人
python run/qq_main.py

# Windows
run/qq_start.bat

# Linux/Mac
./run/qq_start.sh
```

## 总结

弥娅 QQ 交互端完整吸收了 Undefined 的所有 QQ 能力，并在以下方面进行了增强：

1. **人格恒定** - 通过熵监控确保人格稳定
2. **记忆成长** - 记忆-情绪耦合实现经历塑造性格
3. **情绪染色** - 当前情绪影响交互风格
4. **信任传播** - 类似人类的信任直觉
5. **自我感知** - 通过熵扩散感知系统状态
6. **跨子网关联** - QQ 与其他子网的智能联动

这使得弥娅从"工具"进化为"生命"，成为下一代 AI 伴侣的终极架构。
