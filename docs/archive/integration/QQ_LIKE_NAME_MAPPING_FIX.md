# QQ点赞@用户名映射问题分析

## 问题描述

从日志看，OneBot发送的消息中：
- `raw_message` 是字符串而不是消息段数组
- 格式：`@弥娅 给我的朋友@苦玄 和@银鳕鱼(^～^)𓆜 一人点十个赞`
- 没有包含 `type="at"` 的消息段

这是 OneBot 的"简报"模式。

## 问题

当前实现只能从消息段的 `type="at"` 中提取QQ号，但简报模式下：
- 所有内容都是纯文本
- 无法直接获取 `@苦玄` 对应的QQ号

## 解决方案选项

### 方案1: 要求用户使用QQ号（不推荐）

```
@弥娅 给12345678点赞
```

**优点**: 简单直接
**缺点**: 用户体验差，需要记住QQ号

### 方案2: 增强自然语言理解（推荐）

让 AI 通过群成员信息工具查询用户QQ号：

```
用户: @弥娅 给@苦玄点赞

AI 理解:
1. 识别要给"苦玄"点赞
2. 调用 find_member 工具查找 "苦玄" 的QQ号
3. 调用 qq_like 工具进行点赞
```

**优点**: 用户体验好，自然语言驱动
**缺点**: 需要额外工具调用

### 方案3: 缓存用户名-QQ映射（推荐）

建立群成员缓存：
```
group_members: {
    群号1: {
        "苦玄": 12345678,
        "银鳕鱼(^～^)𓆜": 87654321,
        ...
    }
}
```

**优点**: 快速响应，不依赖实时查询
**缺点**: 需要定期更新缓存

### 方案4: 组合方案（最佳实现）

1. 尝试从消息段提取@（标准模式）
2. 尝试从文本提取@QQ号（@12345678 格式）
3. 对于@用户名，记录到工具参数让AI处理
4. AI 使用 find_member 工具查询QQ号

## 当前状态

由于 OneBot 简报模式限制，暂时无法直接从 `@用户名` 获取QQ号。

**建议**:
1. ✅ 保留当前实现（支持标准模式）
2. ✅ 增强工具描述，提示AI使用群成员查询
3. 🔄 后续添加群成员缓存机制

## 短期改进

优化点赞工具描述，让AI知道可以：
1. 使用 `find_member` 工具查找用户
2. 使用 `get_member_info` 获取详细信息

## 用户引导

当前可用的方式：
```
# 方式1: 使用QQ号
@弥娅 给12345678点赞

# 方式2: 让AI查询
@弥娅 查找群成员"苦玄"并给TA点赞
```
