## 🔧 如何使用

### 清理端口（重要！）

每次启动前运行：
```bash
cleanup_port.bat
```

这会清理所有占用端口的进程。

### 启动应用

1. 运行 `cleanup_port.bat` 清理端口
2. 选择选项4：启动桌面UI (Electron)
3. 等待启动完成

### 打开桌宠

1. 在主窗口左侧工具栏点击 "🐾 桌宠" 按钮
2. 桌宠窗口会独立打开
3. 窗口右上角有 "🎭 换装" 按钮

### 切换模型

1. 点击桌宠窗口的 "🎭 换装" 按钮
2. 从下拉菜单选择模型：
   - 御姐猫猫（8个表情）
   - 修女（32个表情）
3. 模型会自动重新加载

### 控制表情

在主聊天窗口：
1. 点击左侧 "Live2D" 按钮
2. 在控制面板中选择表情：
   - 开心（index 0）- 设置 Param2=1 (脸黑)
   - 害羞（index 1）- 设置 Param4=1 (眼泪)
   - 生气（index 2）- 设置 Param3=1 (爱心)
   - 悲伤（index 3）- 设置 Param6=1 (爱心 粉)
   - 平静（index 4）- 设置 Param=1 (脸红)
   - 兴奋（index 5）- 设置 Param7=1 (嘘)
   - 调皮（index 6）- 设置 Param8=1 (话筒)
   - 嘘声（index 7）- 设置 Param74=1 (耳朵消失)

## 🐛 调试方法

### 查看控制台日志

1. 按 `F12` 打开开发者工具
2. 查看Console标签
3. 寻找以下日志：
   ```
   [Live2D] ✅ 通过索引设置表情成功: X
   [Live2D] 设置参数 ParamX = Y
   ```

### 运行诊断脚本

1. 复制 `debug_live2d.js` 的内容
2. 在控制台中粘贴并运行
3. 查看诊断结果

### 手动测试表情

在控制台中运行：
```javascript
// 获取模型
const canvas = document.querySelector('canvas')
const model = canvas.__pixi_app.stage.children[0]
const internal = model.internalModel

// 设置表情（索引0-7）
internal.motionManager.expressionManager.setExpression(0)

// 查看表情列表
console.log(internal.motionManager.expressionManager.expressions)
```

## 📊 关于表情效果

### 为什么看不到明显的表情变化？

可能的原因：

1. **表情类型不同**：这个模型的表情可能不是面部表情，而是装饰效果
2. **参数变化不明显**：某些参数的效果可能很微妙
3. **需要动画时间**：表情变化可能需要几秒钟才能完成
4. **模型设计**：模型本身可能没有明显的表情变化设计

### 如何验证表情是否生效？

1. 查看控制台日志 - 应该显示"设置成功"
2. 观察模型细节 - 留意爱心、眼泪、脸红等效果
3. 运行诊断脚本 - 查看参数值变化
4. 尝试不同的表情 - 比较不同表情的差异

## 🎯 下一步建议

### 如果表情效果不明显：

1. **尝试不同的模型** - 修女模型可能有更明显的表情
2. **调整参数值** - 修改表情文件中的数值
3. **查看模型文档** - 了解模型的具体表情定义
4. **使用其他Live2D模型** - 寻找有更好表情的模型

### 如果端口一直被占用：

1. 运行 `cleanup_port.bat`
2. 重启电脑
3. 检查是否有其他程序在使用这些端口
