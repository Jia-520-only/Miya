## 二、核心模块详细架构

### 2.1 PCUINet 子网架构

```
PCUINet (webnet/pc_ui.py)

│   └── plugins (插件管理)
│
├── 对话系统
│   ├── handle_user_message()

│   └── build_conversation_context()
│
├── 群聊系统
│   ├── create_group()
│   ├── handle_group_message()
│   └── group_sessions (群聊会话)
│
├── 笔记系统
│   ├── create_note()
│   ├── search_notes()
│   └── notes (笔记存储)
│
├── 媒体控制
│   ├── music_control()
│   ├── music_playlist
│   └── current_track
│
├── 画布系统
│   ├── update_canvas()
│   └── canvas_states
│
├── 插件系统
│   ├── load_plugins()
│   ├── execute_plugin()
│   └── plugin_manifests
│
└── 任务调度
    ├── schedule_task()
    └── scheduled_tasks
```

### 2.2 记忆系统架构

```
MemoryEngine (hub/memory_engine.py)
├── 存储接口
│   ├── store() → Redis / Milvus
│   ├── retrieve() → 向量搜索
│   └── update() → 增量更新
│
├── 记忆类型
│   ├── 潮汐记忆 (Tide Memory)
│   │   ├── Redis (高频)
│   │   └── Milvus (长期)
│   ├── GRAG记忆 (五元组)
│   │   ├── 主体-动作-对象-上下文-时间
│   │   └── Neo4j知识图谱
│   └── 梦境压缩
│       ├── 离线去重
│       └── 关联强化
│
├── 检索策略
│   ├── 语义向量搜索
│   ├── 关键词匹配
│   ├── 时间窗口过滤
│   └── 上下文相关性
│
└── 情绪耦合
    ├── 情绪 → 记忆权重
    ├── 回忆 → 情绪唤醒
    └── 记忆巩固 → 人格微调
```

### 2.3 M-Link 传输架构

```
MLinkCore (mlink/mlink_core.py)
├── 五流分发
│   ├── 指令流 (Control Flow)
│   │   └── 内核/中枢 → 执行节点
│   ├── 感知流 (Perception Flow)
│   │   └── 感知层 → 中枢/子网
│   ├── 同步流 (Sync Flow)
│   │   └── 子网 ↔ 子网横向互通
│   ├── 信任流 (Trust Flow)
│   │   └── 信任评分与传播
│   └── 记忆流 (Memory Flow)
│       └── 记忆读写请求
│
├── 路由策略
│   ├── 动态路径评分
│   ├── 优先级通道
│   └── 断链自愈
│
└── 消息格式
    ├── Message (统一消息)
    ├── MessageType (类型)
    └── FlowType (流类型)
```

## 三、数据流图

### 3.1 对话流程

```
用户输入
   ↓
[PC UI] 捕获输入
   ↓
WebSocket / REST API
   ↓
[FastAPI] 接收请求
   ↓
[PCUINet] handle_user_message()
   ↓
   ├─→ 获取/创建会话
   ├─→ 情绪感知 (EmotionManager)
   ├─→ 记忆检索 (MemoryEngine)
   └─→ 构建上下文
        ↓
   [M-Link] 发送控制流
        ↓
   [Hub] 决策引擎
        ↓
   [Memory] 检索相关记忆
   [Emotion] 情绪染色
        ↓
   [LLM] 生成回复
        ↓
   [Hub] 后处理
   [Memory] 存储对话
   [Emotion] 更新情绪
        ↓
   [M-Link] 返回响应
        ↓
   [PC UI] 显示消息
        ↓
   用户看到回复
```

### 3.2 群聊流程

```
用户输入群聊消息
   ↓
[PCUINet] handle_group_message()
   ↓
获取群组和话题
   ↓
遍历群组中的所有Agents
   ↓
for each agent in group.agents:
   [M-Link] → [Hub] → [LLM]
   生成该Agent的回复
   ↓
收集所有回复
   ↓
返回给用户
```

### 3.3 笔记流程

```
用户创建笔记
   ↓
[PCUINet] create_note()
   ↓
存储到 notes 字典
   ↓
[MemoryEngine] store()
   ├─→ Redis (近期)
   └─→ Milvus (向量)
        ↓
用户搜索笔记
   ↓
[MemoryEngine] retrieve()
   ├─→ 向量搜索
   └─→ 语义匹配
        ↓
返回相关笔记
```

## 五、数据库设计

### 5.1 Redis数据结构

```
# 潮汐记忆 (TTL: 3600s)
tide:memory:{session_id}:{timestamp}
├─→ content: "用户: ... \n弥娅: ..."
├─→ emotion: {"type": "joy", "intensity": 0.8}
└─→ tags: ["conversation", "pc_ui"]

# 会话状态
session:{session_id}
├─→ agent_id: "miya_default"
├─→ created_at: "2026-02-28T10:00:00"
└─→ last_active: "2026-02-28T10:30:00"

# 情绪状态
emotion:current
└─→ {"type": "joy", "intensity": 0.8, ...}
```

### 5.2 Milvus集合结构

```
Collection: miya_memory
├─ id: int64 (PK)
├─ content: string
├─ embedding: vector[1536] (OpenAI embeddings)
├─ tags: string[]
├─ emotion: string
├─ timestamp: int64
└─ metadata: json

Index:
├─ IVF_FLAT (embedding)
└─ InvertedIndex (tags)
```

### 5.3 Neo4j知识图谱

```
(:Subject)-[:PERFORMED]->(:Action)-[:ON]->(:Object)
     │                         │
     └──────[:IN_CONTEXT]─────┘
              ↓
          (:Context)
              │
              [:AT_TIME]
              ↓
          (:Time)

Example:
(:弥娅)-[:PERFORMED]->(:回答)-[:ON]->(:用户问题)
     │                         │
     └──────[:IN_CONTEXT]─────┘
              ↓
          (:对话上下文)
              │
              [:AT_TIME]
              ↓
          (:2026-02-28T10:00:00)
```
