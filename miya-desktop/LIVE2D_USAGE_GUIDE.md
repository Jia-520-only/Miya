# Live2D 独立窗口使用指南

## 🎯 重要说明

**当前看到 Live2D "粘在 UI 上"是正常的！** 因为你还没有打开独立窗口。

独立窗口功能已经实现，需要手动激活。

---

## 📖 正确使用流程

### 步骤 1: 启动应用

```bash
cd miya-desktop
npm run dev
npm run dev:electron
```

或者使用：
```bash
test_live2d_standalone.bat
```

### 步骤 2: 进入聊天界面

应用启动后，进入聊天界面。

### 步骤 3: 找到 Live2D 控制面板按钮

在聊天界面左侧工具栏，找到 **机器人图标** 🤮

位置：
```
┌─────────────────────────┐
│  会话列表               │
├─────────────────────────┤
│  [主题切换]             │
│  [用户]      ← 点击这个  │
│  [🤮]  ← 点击这个!     │  ← Live2D控制
└─────────────────────────┘
```

### 步骤 4: 打开 Live2D 控制面板

点击机器人图标 🤮，控制面板将从右侧滑出。

### 步骤 5: 创建独立窗口（关键步骤！）

在控制面板中，点击 **"打开窗口"** 按钮。

**点击后，将会在桌面上创建一个新的 Live2D 窗口！**

### 步骤 6: 现在可以拖动了！

新的 Live2D 窗口出现后：
- ✅ 可以在桌面上自由拖动
- ✅ 可以调整大小
- ✅ 可以设置为置顶
- ✅ 背景是透明的

---

## 🎮 独立窗口 vs 嵌入窗口

### 嵌入窗口（当前看到的）
```
┌─────────────────────────────┐
│ 主应用窗口                  │
│ ┌─────────┬───────────────┐│
│ │聊天区域 │  Live2D      ││  ← Live2D 在这里
│ │         │  (固定位置)   ││     不能拖动
│ └─────────┴───────────────┘│
└─────────────────────────────┘
```

**特点**：
- ❌ 不能拖动
- ❌ 固定在聊天界面中
- ❌ 受主窗口限制

### 独立窗口（需要打开）
```
┌─────────────────────┐  ┌──────────┐
│ 主应用窗口          │  │ Live2D   │  ← 独立窗口
│ ┌───────────────┐  │  │ (可拖动) │     可以拖动！
│ │聊天区域       │  │  │          │
│ │               │  │  │          │
│ └───────────────┘  │  └──────────┘
└─────────────────────┘
```

**特点**：
- ✅ 可以在桌面任意位置
- ✅ 可以自由拖动
- ✅ 可以调整大小
- ✅ 可以置顶显示

---

## 🔍 如何确认是否使用了独立窗口？

### 检查 1: 窗口数量
- **独立窗口**: 你会看到 2 个窗口（主窗口 + Live2D 窗口）
- **嵌入窗口**: 只有 1 个窗口（主窗口）

### 检查 2: Live2D 位置
- **独立窗口**: Live2D 在单独的小窗口中，可以在桌面上拖动
- **嵌入窗口**: Live2D 在聊天界面的右侧

### 检查 3: 控制面板状态
- **独立窗口已打开**: 控制面板显示"窗口已打开"，可以调整大小和表情
- **未打开独立窗口**: 控制面板显示"准备就绪"

## 💡 常见错误

### 错误 1: "为什么 Live2D 还在聊天界面里？"
**原因**: 没有打开独立窗口

**解决**:
1. 点击左侧工具栏的机器人图标 🤮
2. 在控制面板中点击"打开窗口"
3. 等待新窗口出现

### 错误 2: "点击'打开窗口'没有反应"
**原因**:
1. 库文件未加载完成
2. 窗口已在后台

**解决**:
1. 等待 2-3 秒后重试
2. 检查任务栏是否有新窗口
3. 查看控制台错误信息

### 错误 3: "独立窗口打不开"
**原因**: 缺少库文件或初始化失败

**解决**:
1. 确认 `public/libraries/` 下有库文件
2. 检查控制台错误
3. 尝试重启应用

## 🎨 独立窗口功能演示

### 基础操作
1. **拖动**: 点击窗口任意位置并拖动
2. **调整大小**: 拖动窗口边缘
3. **最小化**: 点击窗口最小化按钮
4. **关闭**: 点击控制面板的"关闭窗口"按钮

### 高级操作
1. **设置置顶**: 在控制面板点击"窗口置顶"
2. **精确大小**: 使用控制面板的滑块调整
3. **快捷预设**: 点击"小窗口"、"中窗口"等按钮
4. **表情控制**: 点击控制面板的表情按钮

## 🆘 故障排除

### 问题: 控制台显示 "Could not find Cubism 2 runtime"

**原因**: 这是正常的警告，因为我们使用的是 Cubism 4

**解决**: 无需处理，Cubism 4 模型可以正常加载

### 问题: Live2D 显示空白

**原因**: 模型文件路径错误或加载失败

**解决**:
1. 检查 `/live2d/ht/ht.model3.json` 是否存在
2. 查看控制台错误信息
3. 确认 `public/live2d/ht/` 目录有模型文件

### 问题: 独立窗口无响应

**原因**: 窗口可能在后台或被遮挡

**解决**:
1. 检查任务栏是否有 Live2D 窗口
2. 点击任务栏图标显示窗口
3. 尝试关闭后重新打开

## 📞 需要帮助？

1. **检查控制台错误** - F12 打开开发者工具查看

4. **重启应用** - 有时重启可以解决临时问题
