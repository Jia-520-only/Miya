# MIYA 桌宠使用指南

## 什么是桌宠？

桌宠是一个 Live2D 角色，可以漂浮在桌面上，与您进行互动。它具有以下特点：

- ✅ **无边框窗口**：完全透明，只有 Live2D 模型可见
- ✅ **始终置顶**：显示在所有其他窗口之上
- ✅ **可拖动**：按住 Shift 键拖动桌宠窗口
- ✅ **可缩放**：使用鼠标滚轮调整模型大小
- ✅ **表情控制**：通过主窗口的控制面板更改表情
- ✅ **动作控制**：播放不同的动作

## 如何启动桌宠

### 方法 1：使用完整的启动脚本（推荐）

1. 双击 `start_all.bat`
2. 等待 Vite 服务器启动（约 10 秒）
3. 两个窗口会自动打开：
   - Vite 开发服务器窗口
   - MIYA 主应用窗口
4. 在主应用中点击"桌宠模式"按钮

### 方法 2：手动启动

1. 双击 `start_dev.bat` 启动 Vite 服务器
2. 等待服务器启动完成
3. 在新的终端窗口中运行：`npm run dev:electron`
4. 在主应用中点击"桌宠模式"按钮

## 桌宠控制

### 基本操作

| 操作 | 方法 |
|------|------|
| 移动模型 | 直接拖动 Live2D 模型 |
| 缩放模型 | 使用鼠标滚轮 |
| 移动窗口 | 按住 Shift 键 + 拖动 |
| 最小化窗口 | 点击右上角的最小化按钮（-） |
| 关闭窗口 | 点击右上角的关闭按钮（×） |

### 表情和动作控制

在主应用的侧边栏中：

1. **表情控制**：点击表情按钮更改 Live2D 表情
   - 可用表情：开心、害羞、生气、悲伤、平静、兴奋、调皮、嘘声

2. **动作控制**：点击动作按钮播放动作
   - 可用动作：动作1-动作5

**注意**：表情和动作会同时应用到主窗口和桌宠窗口。

## 故障排除

### 桌宠窗口没有打开

**可能原因**：
- `preload.js` 文件未生成
- 桌宠窗口已经在后台运行

**解决方法**：
1. 检查 `dist-electron\preload.js` 是否存在
2. 如果不存在，删除 `dist-electron` 文件夹并重新启动
3. 检查 Electron 窗口的控制台是否有错误

### 背景不透明

**可能原因**：
- 窗口透明配置未生效
- CSS 样式覆盖了透明设置

**解决方法**：
1. 确保窗口配置中 `transparent: true`
2. 检查 Live2DStandalone.vue 的样式是否设置了 `background: transparent`

### 无法拖动窗口

**解决方法**：
- 确保按住 **Shift 键** 再拖动
- 直接拖动 Live2D 模型可以移动模型本身

### 表情/动作不生效

**可能原因**：
- 模型不支持表情/动作
- IPC 通信失败

**解决方法**：
1. 查看控制台日志中的"可用表情"和"可用动作组"
2. 确认 `electronAPI` 已正确加载
3. 尝试重新打开桌宠窗口

### preload.js 错误

**错误信息**：`Unable to load preload script: preload.js`

**解决方法**：
```bash
# 删除旧的编译文件
rd /s /q dist-electron

# 重新启动
start_all.bat
```

## 技术说明

### 窗口配置

桌宠窗口使用以下配置：

```typescript
{
  width: 280,
  height: 350,
  frame: false,        // 无边框
  transparent: true,   // 透明背景
  alwaysOnTop: true,  // 始终置顶
  skipTaskbar: true,   // 不显示在任务栏
  show: false,         // 页面加载完成后再显示
  backgroundColor: '#00000000'  // 完全透明
}
```

### IPC 通信

主窗口和桌宠窗口之间通过 IPC 通信：

- `live2d:setExpression` - 发送表情控制消息
- `live2d:setMotion` - 发送动作控制消息

### Live2D 模型

当前使用 `ht` 模型：
- 路径：`/live2d/ht/ht.model3.json`
- 表情：8个（expression1-8.exp3.json）
- 动作：无

## 进阶功能

### 自定义桌宠位置

您可以在代码中修改桌宠窗口的初始位置：

```typescript
// electron/modules/live2d-window.ts
const LIVE2D_WINDOW_CONFIG = {
  x: 100,  // X 坐标
  y: 100,  // Y 坐标
  // ...
}
```

### 禁用鼠标穿透

如果希望桌宠窗口不响应鼠标事件（让鼠标穿透到桌面）：

```typescript
live2dWindow.setIgnoreMouseEvents(true, { forward: true })
```

### 添加更多模型

将 Live2D 模型文件放到 `public/live2d/` 目录下，然后修改模型路径。

## 注意事项

1. **开发环境**：桌宠功能只在开发环境中可用
2. **性能**：桌宠窗口会在后台持续运行，可能会影响性能
3. **透明度**：Windows 10/11 支持更好的透明效果
4. **安全警告**：Electron 的安全警告在开发环境中可以忽略，打包后会消失
