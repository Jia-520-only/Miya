## 🎭 简介

MIYA 桌面应用支持 Live2D Cubism 4.x 格式的 3D 动态角色模型。Live2D 技术可以让 2D 插画呈现出生动的 3D 效果，支持表情变化、动作动画等。

### 主要功能

- ✅ 多模型切换
- ✅ 表情控制（8 种基础表情）
- ✅ 动作播放
- ✅ 桌面宠物模式
- ✅ 拖拽移动和缩放
- ✅ 鼠标交互响应

## 🚀 如何使用模型

### 1. 在聊天窗口中查看模型

1. 启动 MIYA 桌面应用
2. 进入聊天界面，模型会自动显示在左侧边栏
3. 模型会根据对话内容自动变化表情

### 2. 打开 Live2D 控制面板

1. 点击左侧边栏的「表情控制」按钮
2. 弹出控制面板后，可以：
   - 切换不同模型
   - 手动选择表情
   - 播放动作
   - 打开桌面宠物模式

### 3. 切换模型

在控制面板的「模型选择」下拉菜单中选择想要使用的模型：
- 系统会自动加载新模型
- 加载过程中显示进度提示
- 加载完成后模型会自动显示

### 4. 调整模型大小和位置

- **缩放**：使用鼠标滚轮缩放模型大小
- **移动**：按住鼠标左键拖拽模型移动

## ➕ 添加新模型

### 方法一：手动配置（推荐）

#### 步骤 1：准备模型文件

确保你的 Live2D 模型包含以下文件：
```
your-model-folder/
├── your-model.model3.json    # 必需：模型定义文件
├── your-model.moc3           # 必需：模型数据文件
├── your-model.physics3.json  # 推荐：物理效果文件
└── expression*.exp3.json     # 可选：表情文件
```

#### 步骤 2：复制模型文件

将模型文件夹复制到项目根目录的 `live2d` 文件夹中：

```
D:/AI_MIYA_Facyory/MIYA/Miya/live2d/your-model-folder/
```

#### 步骤 3：编辑配置文件

打开 `miya-desktop/src/config/live2dModels.ts`，在 `LIVE2D_MODELS` 对象中添加新模型：

```typescript
'your-model': {
  id: 'your-model',                    // 模型唯一标识
  name: '你的模型名称',                // 显示名称
  path: '/live2d/your-model-folder/your-model.model3.json',  // 模型文件路径
  description: '模型描述文字'         // 模型描述
},
```

#### 步骤 4：重启应用

重新启动 MIYA 桌面应用，新模型就会出现在控制面板的模型选择列表中。

### 方法二：自动扫描（开发中）

未来版本将支持自动扫描 `live2d` 文件夹并自动识别模型，无需手动配置。

## 🎬 动作控制

### 播放动作

1. 打开 Live2D 控制面板
2. 在「动作控制」区域选择动作按钮
3. 点击后模型会播放对应的动作动画

### 动作组

动作按照组进行分类，常见的动作组包括：
- `Idle`：待机动作
- `TapBody`：点击身体
- `TapFace`：点击脸部
- `Shake`：摇晃
- 等等...

每个模型可能有不同的动作组，具体取决于模型创作者的设计。

## 🖥️ 桌面宠物模式

### 打开桌面宠物

1. 打开 Live2D 控制面板
2. 点击「打开桌宠」按钮
3. 模型会在独立窗口中显示，可以拖拽到屏幕任意位置

### 桌面宠物特性

- **透明背景**：桌面宠物背景透明，不影响桌面美观
- **鼠标穿透**：点击桌面宠物时不会被遮挡，方便操作其他窗口
- **模型交互**：点击模型时取消鼠标穿透，方便拖拽和交互
- **拖拽移动**：按住鼠标左键拖拽移动宠物位置
- **滚轮缩放**：使用鼠标滚轮调整宠物大小

### 关闭桌面宠物

点击桌面宠物窗口的关闭按钮，或返回控制面板点击「关闭桌宠」按钮。

## ❓ 常见问题

### 1. 模型加载失败

**问题**：点击模型后显示加载错误或黑屏

**解决方案**：
- 检查模型文件路径是否正确
- 确认所有必需文件都已放置到位（.model3.json, .moc3）
- 检查浏览器控制台是否有错误信息
- 确认模型是 Cubism 4.x 格式

### 2. 模型切换时出错

**问题**：切换模型时出现 WebGL 错误或黑屏

**解决方案**：
- 这是正常的，系统会自动重新初始化
- 如果持续出错，请重启应用
- 检查模型文件是否完整

### 3. 表情不生效

**问题**：选择表情后模型没有变化

**解决方案**：
- 检查模型是否包含对应的表情文件
- 确认表情文件名符合标准格式（expression*.exp3.json）
- 部分模型可能没有预定义表情

### 4. 动作不播放

**问题**：点击动作按钮后没有反应

**解决方案**：
- 检查模型是否包含动作文件
- 确认模型支持该动作组
- 查看浏览器控制台是否有错误信息

### 5. 桌面宠物无法打开

**问题**：点击「打开桌宠」按钮没有反应

**解决方案**：
- 检查 Electron 主进程是否正常运行
- 查看应用日志中的错误信息
- 尝试重启应用

### 6. 模型显示位置不正确

**问题**：模型显示偏移或裁剪

**解决方案**：
- 使用鼠标拖拽调整位置
- 使用滚轮调整大小
- 如果模型文件有问题，可能需要联系模型创作者

## 🔧 技术细节

### 系统架构

```
miya-desktop/
├── src/
│   ├── components/
│   │   └── Live2DFull.vue          # Live2D 组件
│   ├── views/
│   │   └── ChatView.vue            # 聊天界面（包含控制面板）
│   ├── config/
│   │   └── live2dModels.ts         # 模型配置文件
│   └── utils/
│       └── pixi-check.ts          # PIXI 加载检查
└── public/
    └── live2d/                     # Live2D 模型文件夹
        ├── ht/
        ├── 修女/
        └── ...更多模型
```

### 技术栈

- **渲染引擎**：PixiJS 6.5.10
- **Live2D SDK**：pixi-live2d-display (Cubism 4.x)
- **WebGL**：WebGL 2.0
- **框架**：Vue 3 + TypeScript
- **桌面框架**：Electron

### 模型格式要求

- **Cubism 版本**：4.x
- **必需文件**：
  - `.model3.json` - 模型定义文件
  - `.moc3` - 模型数据文件
- **推荐文件**：
  - `.physics3.json` - 物理效果文件
  - `.cdi3.json` - 参数定义文件
- **可选文件**：
  - `expression*.exp3.json` - 表情文件
  - `*.motion3.json` - 动作文件

### 性能优化

- **Canvas 动态重建**：切换模型时动态重建 canvas 元素，避免 WebGL 上下文污染
- **资源清理**：自动清理旧模型的内存和 WebGL 资源
- **延迟加载**：按需加载模型文件，减少初始加载时间

## 📝 配置文件示例

### 完整的模型配置

```typescript
{
  id: 'my-model',
  name: '我的模型',
  path: '/live2d/my-model/my-model.model3.json',
  description: '这是一个自定义模型',
  expressions: [
    { name: '开心', file: 'expression1.exp3.json' },
    { name: '害羞', file: 'expression2.exp3.json' }

    { group: 'Idle', files: ['idle.motion3.json'] },
    { group: 'TapBody', files: ['tap_01.motion3.json', 'tap_02.motion3.json'] }
  ]
}
```
